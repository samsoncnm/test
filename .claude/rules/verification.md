# 验证与指标

> 核心：所有"成功"定义必须有可机械验证的命令输出。

## 原则 5：指标必须机械可验证 (Metrics Must Be Mechanical)

- **验证命令化**：所有"成功"定义必须对应一个 shell 命令，输出 parseable 数字
- **反模式**：禁止"看起来更好"、"大概改进了"等主观判断 — 这会杀死自主迭代循环
- **提取规则**：若当前任务无法构造验证命令，先停下来问用户"如何量化这个改进？"

## 原则 6：验证必须快速 (Verification Must Be Fast)

- **快速优先**：使用最轻量的验证手段（lint > unit test > e2e），慢速验证留到 loop 之后
- **超时感知**：单次验证超过 2 分钟需上报，由用户判断是否继续
