# 自主循环状态机 (Autonomous Loop)

当执行多轮迭代任务时，严格遵循以下状态机：

```
循环 N 次:
  1. 回顾：读取当前状态 + git 历史 + 结果日志
  2. 构思：基于历史选择下一步（优先复用成功模式）
  3. 修改：仅改动 in-scope 文件，一次仅一个逻辑变更
  4. Commit：验证前提交（启用 git revert 能力）
  5. 验证：运行机械验证命令，提取数值指标（所有命令带超时）
  6. 守卫：若有 guard 命令，运行并确认无回归
  7. 决策：
     - 改善 + guard 通过 → 保留，继续
     - 改善 + guard 失败 → revert，重做（最多 2 次）
     - 无改善 → revert，丢弃
     - 崩溃 → 尝试修复（最多 3 次），否则丢弃
     - 任何路径失败超过上限 → 停止，上报（见 artifacts.md 原则 11 失败报告模板）
  8. 记录：追加结果到 results.log（注明成功/失败/放弃 + 原因）
```

> [!WARNING] 若任务非多轮迭代（一次性开发），跳过状态机但仍遵循所有原则。

> [!NOTE] 代码提交前必须通过 linter 和静态代码检查，确保无低级语法错误。
