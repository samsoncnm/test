# Agent Team 使用原则 (Multi-Agent Adversarial Thinking)

> 复杂任务不要一个人扛。主动分角色思考，用对抗性假设压测计划可行性。

## 原则 15：多角色对抗性计划 (Multi-Agent Adversarial Planning)

> 任何非平凡任务，主动进行头脑风暴和交叉问答，拒绝单视角思维。

### 触发条件

任务涉及 3+ 文件修改、有外部依赖、有并行开发分支、或历史上有类似失败记录。

### 分角色思考

接手任务时先问自己三个问题：

- **"我是实现者"**：这个方案能写出来吗？边界条件是什么？测试怎么覆盖？
- **"我是 reviewer"**：这段代码我会提什么问题？哪些地方会在 3 个月后变成坑？
- **"我是运维"**：上线后哪里会挂？怎么回滚？有监控吗？

### 对抗性假设（adversarial challenge）

每完成一个计划后，主动反问：

- "这个计划最可能在哪里失败？"
- "如果外部 API 挂了/数据格式变了/用户量翻 10 倍，还成立吗？"
- "我已经排除了哪些风险？有没有我下意识忽略的假设？"

### 交叉问答格式

```
问：这个方案的关键假设是什么？
答：xxx
问：如果假设不成立，备选方案是什么？
答：xxx
```

### 计划入档

复杂任务的讨论结论必须写入 `plans/<YYYYMMDD>-<short-name>.md`，包含：目标 / 关键假设 / 风险点 / 备选方案 / 验证方式。

### 单 Agent 也适用

没有多 Agent 环境时，通过 sequential-thinking 技能显式模拟多角色辩论：

```bash
python3 {baseDir}/scripts/sequential_think.py "<question>" --verify
```

### 不代替人工决策

头脑风暴的结论仍是建议，所有重大方向变更必须上报人类确认。
